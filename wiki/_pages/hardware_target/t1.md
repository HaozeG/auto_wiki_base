# T1

T1 (Torrent-1) is a RISC-V Vector implementation developed by ChipsAlliance, inspired by the Cray X1 vector machine. It is designed as a lane-based microarchitecture with intensive chaining support and SRAM-based vector register files (VRFs). T1 implements the RISC-V Vector extensions Zve32f and Zve32x, and supports configurable VLEN and DLEN up to 64K, allowing for a wide range of performance and area trade-offs. The design is parameterizable and can be integrated with any RISC-V scalar core, using a forked version of the Rocket Core as the default scalar component. T1 is written in Chisel and includes an emulator for verification and design space exploration. The architecture emphasizes chaining between functional units and high memory bandwidth through a configurable load-store unit with instruction-level out-of-order execution.

## Key Claims

- T1 uses a lane-based microarchitecture with 32-bit per lane and support for multiple lanes.
- Full chaining support: Load to Multiple-Exec to Store to Load chaining.
- Configurable VLEN and DLEN up to 64K.
- Supports Zve32f and Zve32x RISC-V Vector extensions.
- Uses a forked Rocket Core as scalar part, but this can be replaced by any RISC-V scalar CPU.
- Load-Store Unit with instruction-level out-of-order execution and configurable outstanding size.
- Design methodology: overall vector core frequency limited by VRF SRAM; bandwidth bottleneck limited by VRF SRAM; LSU bandwidth limited by memory ports.
- Supports MMIO Domain-Specific Accelerators (DSA).
- Bare-metal execution only; no operating system support.
- Verified using an emulator (T1Emulator) with Spike as reference for differential testing.

## Optimization-Relevant Details

- ISA/profile: RISC-V Vector, Zve32f, Zve32x
- Vector/matrix/accelerator support: Lane-based vector execution with configurable number of lanes (32-bit per lane); full chaining between load, multiple-execute, and store; banked SRAM-based VRF with configurable banking; support for widening and narrowing instructions via direct-connected lane interconnection.
- Memory/cache/TLB/DMA: Configurable banked memory port for LSU; instruction-level out-of-order load/store; configurable outstanding size to mitigate memory latency; no MMU support for high-bandwidth ports; no coherence support.
- Compiler/toolchain support: Designed with Chisel hardware description language; verified with Spike RISC-V ISA simulator; bare-metal toolchain using RISC-V GCC.
