# RISC-V Vector Extension Optimization via Probabilistic Programs

Tensor program optimization for the RISC-V Vector Extension (RVV) using probabilistic programs refers to a compiler-based approach that integrates the RVV instruction set into Apache TVM's MetaSchedule framework, an autotuning system that uses probabilistic search to optimize tensor operation mappings for hardware targets. This method addresses the challenge of efficiently deploying AI workloads on RISC-V CPUs with vector units without relying on expert-tuned libraries or autovectorization alone. By modeling the optimization space probabilistically, the approach automatically discovers efficient mappings of neural network operators to RVV instructions, achieving significant latency reductions compared to standard compiler autovectorization and hand-crafted libraries. The technique was evaluated on both FPGA-emulated and commercially available RISC-V SoCs implementing the RVV 1.0 specification, demonstrating consistent performance improvements across a range of AI workloads while also reducing code memory footprint.

## Key Claims

- The approach integrates the RISC-V Vector Extension into TVM's MetaSchedule framework, enabling probabilistic autotuning for tensor programs.
- On FPGA-emulated RISC-V SoCs, the proposal achieves a mean 46% improvement in execution latency compared to GCC's autovectorization and 29% improvement over the muRISCV-NN hand-crafted library.
- On a commercially available RISC-V SoC implementing RVV 1.0, the solution finds mappings that are on average 35% faster than those generated by LLVM.
- The resulting binaries have a smaller code memory footprint, making the approach suitable for embedded devices.
- The authors have open-sourced the proposal to facilitate expansion to other RISC-V extensions.
