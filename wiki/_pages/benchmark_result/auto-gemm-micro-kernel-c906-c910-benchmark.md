# Auto-Generated GEMM Micro-Kernels on C906 and C910

The Auto-Generated GEMM Micro-Kernels on C906 and C910 benchmark evaluates automatically generated micro-kernels for general matrix multiplication (GEMM) on two T-Head RISC-V cores implementing the RISC-V Vector Extension (RVV): the XuanTie C906 and XuanTie C910. The micro-kernels are generated using a Python-based automatic assembly code generator that emits RVV code with a variety of specific and general optimizations, aiming to address performance portability across these cores while reducing developer effort. The benchmark compares these auto-generated micro-kernels against the state-of-the-art tuned BLAS library OpenBLAS on both cores. The results, reported in the SC23 workshop paper by Igual, Piñuel, Catalán, Martínez, Castelló, and Quintana-Ortí, show speed-ups of 3x on the C910 and 1.3x on the C906 relative to OpenBLAS. This benchmark provides evidence that automatic code generation can achieve significant performance improvements on diverse RISC-V vector processors, highlighting the importance of architecture-aware code generation for HPC workloads.

## Key Claims

- Automatically generated GEMM micro-kernels achieve a 3x speed-up over OpenBLAS on the XuanTie C910 core.
- Automatically generated GEMM micro-kernels achieve a 1.3x speed-up over OpenBLAS on the XuanTie C906 core.
- The code generator is written in Python and emits RVV assembly code with configurable optimizations.
- The approach targets performance portability across different RISC-V vector processor implementations (C906 and C910).
